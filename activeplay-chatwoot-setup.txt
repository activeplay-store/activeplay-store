## ПРОБЛЕМА КРАСНОЙ МЕТКИ "НЕ УДАЛОСЬ ОТПРАВИТЬ"

### Что происходит:
- Оператор пишет ответ в Chatwoot → webhook идёт на /chatwoot-webhook → мост отправляет в VK → клиент получает сообщение
- НО Chatwoot показывает "Не удалось отправить" потому что API inbox не получает подтверждение доставки

### Что попробовать:
1. Поменять URL вебхука в настройках источника VK ActivePlay (шестерёнка → Настройки → URL вебхука) на `https://chat.activeplay.games/chatwoot-webhook`
2. Если не помогает — нужно в скрипте после отправки в VK делать обратный вызов Chatwoot API для обновления статуса сообщения
3. Возможно нужно использовать Chatwoot API endpoint для создания outgoing message вместо стандартного Reply
