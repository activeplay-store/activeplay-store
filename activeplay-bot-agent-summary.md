## ФОРМУЛА СКОРИНГА СКИДОК (для карусели главной)

```
БАЛЛ = (hypeScore × 0.40) + (discountScore × 0.30) + (savingScore × 0.20) + (freshness × 0.10)
```

- **hypeScore** (1-10): RAWG ratingsCount → 5000+=10, 2000+=7, 500+=5, 100+=3. Бонус MC 85+=7, 90+=10. Fallback: brandFallback словарь (45 брендов)
- **discountScore**: 65%+=10, 40%+=7, 20%+=4
- **savingScore**: 7000₽+=10, 4000₽+=7, 2000₽+=5, <2000₽=3
- **freshness** (1-10): <12м=10, 1-2г=7, 2-4г=5, 5+=3

Градация бейджей: 🟢 20-39%, 🟡 40-64%, 🔴 65-90%

## СТРАНИЦА /sale (техдок)

- 4 зоны: 🔴 Суперцена 65-90%, 🟡 Большая 40-64%, 🟢 Скидка 20-39%, ⚡ Свежие <20%
- Сортировка внутри зон: по экономии в рублях
- PS Plus цены определяют зону (FC26: PS Plus -69% → красная зона)
- Без лимитов по количеству — всё что прошло фильтр
- Файл данных: `src/data/deals.ts`
- Агент будет писать только в deals.ts, не трогать компоненты

## ВАЖНЫЕ ГРАБЛИ

1. **scp перезаписывает .env** — ВСЕГДА после scp: `sed -i 's|/home/activeplay/activeplay-store/agent/data|/var/www/activeplay-store/agent/data|' /home/activeplay/agent-bot/.env`
2. **git push перезаписывает правки на сервере** — НЕ править через nano на сервере файлы агента (git затрёт). Только через Claude Code → git push
3. **Telegram лимит 4096 символов** — разбивать сообщения на части по 10 игр
4. **RAWG не знает свежие/спортивные игры** — brandFallback словарь как страховка
5. **Парсинг 2 часа** — 4 категории × 2 региона. Оптимизация нужна
6. **TR ≠ UA** — разные скидки, разные игры, разные цены. На странице /sale нужны два отдельных списка
7. **Bash ломается на `!` в двойных кавычках** — использовать одинарные кавычки для node -e
